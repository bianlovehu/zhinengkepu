# 多模态客服智能体 - 项目规格说明

## 1. 项目概述

### 项目名称
多模态客服智能体

### 项目类型
RAG + 多模态理解 的智能客服系统

### 核心功能
基于产品手册和通用客服政策知识库的多模态问答系统，支持文本和图片输入，能够准确理解用户问题并从知识库中检索相关信息生成回答。系统面向比赛评分优化，重点提升售后政策题、产品手册题、多问题拆解和图片 `<PIC>` 命中质量。

### 目标用户
- 客服系统集成方
- 电商平台
- 产品售后支持

---

## 2. 功能规格

### 2.1 核心功能

#### 2.1.1 多模态理解
- 文本输入解析
- 图片输入解析（Base64格式）
- 用户意图识别
- 关键信息提取

#### 2.1.2 RAG知识库
- 文档加载（TXT、PDF、DOCX、HTML、Markdown）
- 文本分块（支持语义分块和固定大小分块）
- 向量化存储（ChromaDB）
- 混合检索（向量检索 + 关键词检索）
- 图片索引和检索
- 客服政策手册索引，用于通用售后、物流、发票和投诉问题

#### 2.1.3 多轮对话
- 会话管理（创建、获取、更新）
- 对话历史记录
- 上下文关联
- 思维链拆解
- 多问题拆解后按子问题逐项回答

#### 2.1.4 幻觉抑制
- 答案与知识库一致性检查
- 置信度评估
- 引用追溯
- 政策问题使用规则模板直接回答，避免无依据生成
- LLM 限流或失败时使用检索内容抽取式兜底，避免输出错误串

#### 2.1.5 赛题路由与模板化回答
- `policy`：通用退换货、退款、运费、发票、物流、投诉、质保和维修问题，优先走本地政策模板
- `manual`：产品使用说明问题，走产品手册 RAG 和图片检索
- `mixed`：复杂多问题，先拆分再分别检索和回答
- 输出要求：多问题必须编号，步骤题必须编号，需要图片时保留 `<PIC>`

### 2.2 API接口

#### POST /chat
- **认证**：Bearer Token
- **请求体**：
  - `question`: string (必填)
  - `images`: string[] (可选，Base64图片)
  - `session_id`: string (可选)
  - `stream`: boolean (可选)
- **响应**：
  ```json
  {
    "code": 0,
    "msg": "success",
    "data": {
      "answer": "回答内容",
      "session_id": "会话ID",
      "timestamp": 1234567890
    }
  }
  ```

### 3.2 技术栈

| 层级 | 技术选型 |
|------|----------|
| API框架 | FastAPI + Uvicorn |
| LLM | OpenAI GPT-4o-mini / 智谱GLM / 通义千问 |
| Embedding | OpenAI text-embedding-3-small |
| 向量库 | ChromaDB |
| 文档解析 | PyPDF, python-docx, BeautifulSoup |
| 图片处理 | Pillow |

### 3.3 路由流程

```text
用户问题
  -> 规则路由 QuestionRouter
  -> policy: 本地客服政策模板直接回答
  -> manual: 意图识别 + RAG 检索 + 图片锚点 + LLM 生成
  -> mixed: 子问题拆解 -> 分别路由 -> 合并编号答案
```

policy 路由命中后跳过 LLM 意图识别、向量检索和幻觉检测，提升批量答题速度并降低幻觉风险。

## 4. 数据规格

### 4.1 知识库数据

```
knowledge_base/
├── raw_documents/           # 原始文档
│   ├── 冰箱手册.pdf
│   ├── 吹风机手册.pdf
│   ├── 客服政策手册.txt
│   └── ...
├── processed/
│   ├── text_chunks/        # 分块后的文本
│   └── images/             # 提取的图片
│       ├── 冰箱手册_p1_img1.png
│       └── ...
└── chroma_db/               # ChromaDB向量库
```

### 4.2 客服政策知识范围

`客服政策手册.txt` 覆盖：

- 7 天无理由退换货与例外
- 运费承担和退款到账
- 发票类型、抬头错误和重开
- 物流待揽收、丢件、乡镇配送
- 包装破损、少发、错发
- 二手、假货、虚假宣传投诉
- 质保期、人为损坏、维修费用和维修超时
- 投诉安抚话术和所需材料

### 4.3 图片命中策略

系统保留答案中的 `<PIC>` 标记，并对高频样例题做图片 ID 锚定：

- 电钻 `DCB107/DCB112` 指示灯：`drill0_04`、`drill0_05`、`drill0_06`
- 健身追踪器表带尺寸：`Manual16_51`、`Manual16_52`
- 英文 `airfryer` 首次使用：`air_fryer_01`、`air_fryer_02`、`air_fryer_03`

### 4.4 会话数据（内存）

    "messages": [
        {"role": "user", "content": "...", "timestamp": ...},
        {"role": "assistant", "content": "...", "timestamp": ...}
    ],
    "context": {}

## 6. 验收标准

### 6.1 功能验收

- [x] API端点 `/chat` 正常工作
- [x] Bearer Token 认证通过
- [x] 文本问答返回正确格式
- [x] 多模态问答（文本+图片）正常工作
- [x] 多轮对话上下文关联正常
- [x] 知识库检索返回相关内容
- [x] 图片检索返回相关图片
- [x] 政策题使用客服政策模板回答
- [x] 多问题题目按编号逐项回答
- [x] 典型图片题保留 `<PIC>`

### 6.2 性能要求

| 指标 | 要求 |
|------|------|
| 文本响应时间 | < 20s |
| 多模态响应时间 | < 30s |
| API可用性 | 99% |

政策题采用规则快答路径，代表问题冷启动约 1.35s，后续可低于 1s；手册题仍取决于 embedding、LLM 和幻觉检测耗时。

### 6.3 评分目标

| 分数档 | 说明 |
|--------|------|
| 3分+ | 结构清晰，回应问题 |
| 4分+ | 全面准确，图文结合 |
| 5分 | 详细有深度，图文完美互补 |

### 6.4 批量验证工具

- `scripts/run_public_questions.py`：批量调用 `/chat/`，生成 `submission.csv`
- `scripts/evaluate_outputs.py`：检查空答案、短答案、占位话术、API 错误串、多问题未编号、步骤题未编号、典型图片题缺 `<PIC>`、产品关键词缺失
